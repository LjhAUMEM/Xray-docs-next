# Hysteria

Клиентская реализация протокола Hysteria.

Эта страница очень простая, поскольку протокол Hysteria фактически разделен на простой протокол управления прокси и оптимизированный транспортный уровень QUIC. В Xray сам протокол и транспортный уровень разделены. Более подробную информацию (например, о brutal) см. в разделе транспорта  [hysteriaSettings](../transports/hysteria.md).

::: tip
Протокол `hysteria protocol` сам по себе не имеет аутентификации. При использовании транспорта, отличного от `hysteria`, он не сможет проксировать `udp`. Также не рекомендуется использовать его с другими транспортными уровнями.
:::

## OutboundConfigurationObject

```json
{
  "version": 2,
  "address": "192.168.108.1",
  "port": 3128
}
```

> `version`: number

Версия Hysteria, должна быть 2.

> `address`: string

Адрес прокси-сервера Hysteria, обязательный параметр.

> `port`: int

Порт прокси-сервера Hysteria, обязательный параметр.
